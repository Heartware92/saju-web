# compact-prep — 압축 전 핵심 정리 (역순 스캔)

이 스킬이 호출되면, 대화 전체를 **가장 최근 → 과거 순(역순)** 으로 훑으며 아래 형식의 요약을 응답으로 출력한다. 이 요약이 대화 마지막에 남아 있으면 이어지는 `/compact`가 이를 우선 반영한다. 요약 출력 후 사용자에게 `/compact` 실행을 안내한다.

## 역순 스캔 원칙

1. **최신이 우선이다.** 같은 주제가 여러 번 등장하면 가장 마지막 결정만 남긴다 (중간에 뒤집힌 결정의 과거 버전은 버린다).
2. **결과만 남기고 과정은 버린다.** 시행착오·실패 반복·중간 논의는, 별도 문서에 기록돼 있으면 문서 경로 한 줄로 대체하고, 아니면 최종 결론 한 줄로 압축한다.
3. **이미 완료·배포·문서화된 작업은 포인터만.** 커밋 해시, 파일 경로, URL로 줄인다. 코드 세부 내용은 남기지 않는다 (코드는 리포에 있다).
4. **진행 중인 것이 제일 귀하다.** 미완료 작업의 현재 상태와 "바로 다음 액션"은 절대 유실되면 안 된다.

## 출력 형식

```
# 세션 핵심 요약 (compact용 · 최신 우선)

## 1. 진행 중 — 다음 액션 (가장 중요)
- [작업명] 현재 상태 / 바로 다음에 할 일 / 관련 파일·경로

## 2. 이 세션의 확정 결정 (최신 → 과거)
- 결정 내용 한 줄 + 근거 한 줄 (뒤집힌 결정은 최종본만)

## 3. 포인터 (상세는 여기에 있음)
- 커밋: 해시 — 한 줄 설명
- 문서: 경로 — 무엇이 담겼는지
- 페이지/URL: 확인 위치

## 4. 미해결·보류·주의사항
- 사용자 액션 대기 / 알려진 리스크 / 다음 세션에서 주의할 것

## 5. 버려도 되는 것 (compact가 과감히 삭제할 대상)
- 시행착오 과정 상세, 중간 실패 로그, 이미 문서화된 내용의 원문, 완료된 변경의 코드 diff 논의
```

## 마무리

요약 출력 후 한 줄 안내: "이 요약을 기준으로 `/compact`를 실행하세요. 요약에 빠진 중요한 내용이 있으면 먼저 알려주세요."

## 주의

- 요약에 새 정보를 지어내지 않는다 — 대화에 실제로 있었던 내용만.
- 메모리(MEMORY.md)나 work-log.md에 이미 기록된 것은 "기록 완료"로 표시하고 중복 서술하지 않는다.
- 세션이 여러 주제를 오갔으면 주제별로 묶되, 각 주제 안에서 역순 원칙 유지.
