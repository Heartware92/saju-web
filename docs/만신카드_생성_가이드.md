## 1. 확정 표준 레시피 (이대로만 하면 됨)

| 항목 | 값 | 비고 |
|---|---|---|
| 모델 | **V7** | 옴니 레퍼런스가 V7 전용. V8.1은 옴니 없음 |
| 비율 | **2:3 (Portrait)** | 카드 UI가 aspect-[2/3]. 9:16 아님! |
| Weirdness / Variety | **둘 다 0** | 덱 일관성의 기본 |
| Personalize (P) | **끄기** | 개인화 프로필이 그림체를 틀어버림 |
| Style Reference | **확정 마스터 이미지** | 화풍 잠금. `--sw 600` |
| Omni Reference | **같은 마스터 이미지** | 물건/캐릭터 생김새 잠금. `--ow 300` |
| 파라미터 | `--ow 300 --sw 600 --no frame, border, text, letters, characters, watermark, person, face, eyes, tree, plant, stack` | 프롬프트 맨 끝, 영어만 |

**레퍼런스 원칙:**
- 스타일 마스터 = `이미지/만신타로/옥황_스타일마스터.png` (**테두리 크롭판**. 원본은 프레임이 그려져 있어 sref로 쓰면 프레임이 전염됨)
- 엽전 시리즈는 **확정 한 닢**이 마스터 (Style + Omni 동시 장착)
- **체인 금지**: 두 닢을 참조해 세 닢을 뽑는 식으로 직전 결과물을 갈아끼우면 복사본의 복사본이 되어 스타일이 표류함. **항상 같은 마스터 1장에서 출발**
- 신령 캐릭터도 동일: 옥황 마스터 고정. (물건 카드는 그 부류의 확정 1호를 옴니로 추가)

## 2. 프롬프트 공통 템플릿

```
Stylized card art for a Korean shamanic oracle deck, charming storybook illustration style.
[피사체 — EXACTLY N (숫자) 이중 표기 + 배치 묘사].
Seen at eye level, a warm beam of golden light falling from above, golden glow on the ground
beneath them, dark misty background with faint floating sparks.
The coins are INANIMATE OBJECTS - they have NO faces, NO eyes, NO expressions.  ← 사물 카드만
The coin surfaces are smooth and worn with NO characters, NO letters, NO markings engraved on them.
Rich deep color palette with gold-leaf accents. NO frame, NO border.
--ow 300 --sw 600 --no frame, border, text, letters, characters, watermark, person, face, eyes, tree, plant, stack
```

**우리 덱의 장면 문법 (모든 카드 공통 — 이 문장이 세트감의 핵심):**
> Seen at eye level, a warm beam of golden light falling from above, golden glow on the ground beneath them, dark misty background with faint floating sparks.

- 눈높이 시점 + 상단 광선 + 바닥 금빛 발광 + 어두운 안개 배경. 이 4요소가 빠지면 혼자 다른 카드가 됨.
- 신령 캐릭터용으로 바꿀 땐 fair porcelain skin tone, Korean facial features 추가 (당금애기 피부톤 이국적으로 나온 사례).

## 3. 실패 사례집 (원인 → 해결. 전부 실제로 겪은 것)

| # | 증상 | 원인 | 해결 |
|---|---|---|---|
| 1 | 영화 컨셉아트풍 사실주의로 나옴 | **스타일 레퍼런스가 실제로 안 걸림** (업로드만 하고 칩 미장착 / 텍스트 교체 시 칩 증발) | 제출 전 입력창에 이미지 칩 확인. 제출 후 잡 정보에 썸네일·sref 표시 확인. 없으면 그 잡은 버림 |
| 2 | 그림 안에 금색 테두리가 자꾸 생김 | ① "oracle card" 단어 ② **마스터 이미지에 프레임이 그려져 있어서 sref가 프레임까지 학습** | 마스터를 테두리 크롭판으로 교체. `--no frame, border` 유지. 프레임은 UI 오버레이로 분리 |
| 3 | 엽전에 눈코입이 생김 (의인화) | 프롬프트의 **"Cute"** 단어 | "Cute" 제거, "INANIMATE OBJECTS - NO faces" 명문화, `--no face, eyes` |
| 4 | 뭉개진 가짜 한자가 표면에 생김 | "worn embossed surface" + 옛날 동전 학습 기억 | "smooth and worn with NO characters" + `--no text, letters, characters`. **글자(一~六, 카드명)는 그림에 넣지 않고 코드 텍스트 레이어로 렌더** |
| 5 | 탑뷰(부감)로 바뀌어 세트감 깨짐 | "laid flat", "arranged in formation" 류 표현 | 장면 문법 문장(eye level + beam + misty) 강제 삽입 |
| 6 | 엽전 나무가 자라남 | "blooming arrangement(만개한)" | 식물 은유 금지. `--no tree, plant` |
| 7 | 도미노처럼 겹쳐 쌓임 (개수 판독 불가) | "row, leaning against each other" | "each coin fully visible and NOT stacked" + `--no stack` |
| 8 | 개수 틀림 (특히 5·6개) | MJ가 5 이상을 잘 못 셈 | "EXACTLY FIVE (5)" 이중 표기. 4장 중 맞는 컷 선택. 전멸 시 리롤. 1장 초과면 Vary Region으로 지우기 |
| 9 | 작은 엽전이 단추로 변질 | 화면에 작게 그려질수록 옴니 잠금 약화 | 엽전이 크게 그려지는 구도 유지. 작은 조연 엽전은 감수 |
| 10 | Vary Strong이 수정한 프롬프트 무시 | 웹 Vary는 원본 프롬프트로 돎 | Vary로 내용 바꾸지 말 것. **Use → Style + Create 바에 새 프롬프트** 방식 사용 |
| 11 | 파라미터 에러 (Unrecognized parameter) | `--` 뒤에 한국어/특수문자 | 프롬프트 본문 영어만, 파라미터는 맨 끝 |
| 12 | 레퍼런스끼리 궁합 문제 | 프레임 이미지를 레퍼런스로 걸면 금속 부조풍으로 오염 | 프레임은 생성과 완전 분리 (UI 에셋). 레퍼런스는 회화 마스터만 |

## 4. 도구 사용법 (리롤 전에 시도할 것)

1. **Vary Region (부분 수술)** — 이미지 → Edit → 문제 부위만 브러시 → 짧은 프롬프트. 마름모 구멍→"square hole", 가짜 글자→"smooth surface", 초과 엽전→"dark misty ground"로 삭제
2. **Vary Subtle** — 구도 좋은데 디테일 아쉬울 때
3. **Upscale Subtle/Creative** — 최종본 화질 상향
4. **리롤** — 장면 자체가 틀렸을 때만 (나무, 탑뷰, 배경 이탈)

판단 순서: "80점, 한 군데 아쉬움" → Vary Region / "60점" → Vary Subtle / "장면 틀림" → 리롤

## 5. 검수 체크리스트 (4장 중 고를 때, 장당 10초)

1. **개수 정확한가** (엽전패 1순위)
2. 테두리가 안 그려졌는가
3. 글자/뭉개진 한자 흔적 없는가
4. 얼굴(의인화) 없는가
5. 네모 구멍 온전한가 (마름모 변형 체크)
6. **장면 문법 4요소** — 눈높이 / 상단 광선 / 바닥 발광 / 안개 배경
7. 마스터와 같은 화풍인가 (선 굵기·색감·질감)

## 5-1. 구도 안전영역 (프레임·UI 가림 스펙 — 2026-07-12 확정)

프레임 크롭 확정(금선이 카드 가장자리 밀착) 후 실측한 가림 영역. **일러스트 2:3 캔버스 기준 %**.
시각 가이드: `이미지/만신타로/일러스트_안전영역_가이드.png` (빨강=항상 가림, 노랑=요약 카드에서만 가림)

| 영역 | 범위 | 가리는 것 | 적용 |
|---|---|---|---|
| 전 둘레 | 가장자리 ~2.5% | 프레임 금선+자개 밴드 | 전 카드 |
| 상단 중앙 | 폭 30~70% × 높이 0~14% | 매듭+방울 장식(13.2%까지) + 패 라벨 필 | 전 카드 |
| 하단 중앙 | 폭 8~92% × 높이 80~93% | 카드명 텍스트 + 하단 그라데이션 | 풍습·엽전 요약 카드 |
| 좌상단 | (7%, 5%) 지름 ~13% 원 | 한자 인장 뱃지 | 풍습·엽전 요약 카드 |

**구도 규칙:**
- 캐릭터/피사체(모자·머리장식 포함)는 **세로 16~80% 구간** 안에 배치. 상단 15%는 배경(광선·하늘)만
- 가장자리 3%는 잘려도 되는 배경 블리드로 취급
- 프롬프트에 추가할 문구: `generous headroom above the subject, the subject occupies the central 65% of the canvas height`
- 사례: 옥황상제 1호(2026-07-12)는 모자 상단이 13%에 있어 방울 장식과 맞닿음 — 가림은 없지만 답답. 재생성 시 위 규칙 적용
