- **탭(7그룹/13탭)**: 현황(overview) · 회원(members: 활성/탈퇴) · 매출·크레딧(orders/credits) · 서비스 이용(usage/records/consultations) · 고객지원(inquiries) · 분석(analytics 유입·이탈/insights) · 시스템(ops/payments/jobs)
- **집계 표준 패턴**: `requireAdmin → resolveAudience → cached(key+suffix, compute)`, compute 내부 모든 쿼리에 `includeAudience(excludeUsers(q, ex), audience)`
  - `_excluded`: PG심사 테스트계정 ∪ env(ADMIN_EXCLUDED_EMAILS) ∪ admin_excluded_users(DB토글) 제외
  - `_audience`: 글로벌 코호트 필터(성별·연령·세그·가입경로·기간). **비로그인 페이지뷰는 user_id=null이라 필터 시 제외**
  - `_cache`: 30초 TTL + dedup, env 있으면 Upstash Redis 공유. 변경 라우트는 끝에 `invalidateAll()`
  - `_userAggregates`: **회원=user_agreements(약관동의) 보유 기준**. listUsers perPage 1000 단일페이지
  - `_emailMap`: user_id→email 30초 공유 캐시
  - `_audit`: writeAudit → admin_audit_logs (best-effort)
  - `_preservedRevenue`: 탈퇴자 보존 주문을 매출에 복원
- **시간**: KST offset 540분 통일. **크레딧 = 단일 '달(moon)'** (sun 폐지).

## 1. 진행중 / 미해결 (우선순위)

1. **감사로그 기록 실패 (실배포 영향)** — `admin_audit_logs.actor_user_id`가 NOT NULL인데 API키 어드민은 user id가 없어 INSERT 실패. **크레딧조정·정지·메모 등 모든 어드민 액션이 감사로그에 안 남음**(운영 탭 "감사 로그"가 빔). 크레딧조정은 credit_transactions에도 남아 "관리자 크레딧 조정"엔 보임. → `actor_user_id` nullable 마이그레이션으로 수정 제안(사용자 승인 대기).
2. **회원 1000명 스케일 천장** — `auth.admin.listUsers({perPage:1000})` 단일페이지라 1001번째부터 무성 누락(`_userAggregates`·`_emailMap`·`_audience`·ops/insights 영향). 1000 근접 시 페이지네이션+truncated 경고 필요. (현재 ~61명)
3. **휴대폰 미인증자가 회원수에 포함** — 약관동의(user_agreements)가 휴대폰 인증보다 먼저라, 휴대폰 단계 이탈자도 회원으로 집계됨. 필요 시 "휴대폰 미인증" 구분/필터 추가 제안.
4. **preserved 미반영 집계** — 탈퇴자 보존분은 **매출(stats·orders/summary)에만** 합산. 크레딧 흐름·이용 분석엔 미반영(요청 시 동일 방식 추가 가능).
5. **"공유 결과" 라벨 명확화(제안만)** — `/share/[token]` 진입·이탈경로 라벨을 "공유링크 유입(받은 링크)" 등으로 바꿀지 사용자 결정 대기.
6. **회원 이름 컬럼 미노출** — `_userAggregates`에 이름 필드 없음(2026-06-19 조사). 프로필명 우선→카카오 닉네임 폴백 추가 제안 상태.

## 2. 어드민 이력 (시간순, 최신 상단)

### 2026-07-18 — 회계 탭 신설 (계약부채 모델 · FIFO · 정산기록 · 부가세)
- **신규 "회계" 그룹/탭** — 선불 달 크레딧을 계약부채(선수금) 모델로 회계처리하는 전용 탭. 이카운트 일반전표(3차/4대) 형식으로 분개를 바로 옮겨 적는 구조.
- **API**: `accounting/summary`(집계, 60초 캐시) + `accounting/settlements`(정산 입금 CRUD). **마이그 057** `admin_settlements` 테이블 신설(RLS 잠금, service_role 전용) — **선별 적용**: 원격 미적용 대기 중이던 055(KPN)·056(알림톡)은 건드리지 않고 057만 push(`--include-all`, 파일 일시 분리 방식).
- **탭 구성(흐름순)**: KPI → ①매일(일별 충전 분개 10일) → ②월말(월 마감 용역매출 분개 + 정산 입금 기록/수수료 자동 역산) → ③부가세 신고(분기별 과세표준·세액, PG별 분해) → ④검증·조회(FIFO 크레딧 원장 + DB 잔액 자동 대조, 접이식 상세 3종) + 이카운트 CSV.
- **회계 모델(사용자 확정)**: 결제=미수금/계약부채+부가세예수금(결제금액×10/110) · 사용=FIFO 유료 lot 소비분만 용역매출(무료 lot 무분개) · 탈퇴 미사용=낙전 매출 · 수수료=월 세금계산서 실측(부가세대급금 월 1회) · LLM비용=월 청구서.
- **데이터 특이점**: credit_transactions.order_id 전부 null → 패키지 단가는 reason 텍스트 매칭(pricing.ts 재사용). purchase 유형 중 "회원가입 환영 보너스"는 무료로 분류. 탈퇴자 결제는 preserved_transactions 합산 필수.
- **FIFO 버그 수정**: 환불·관리자 회수의 lot 차감이 매출로 인식되던 경로 차단. 반올림 통일(float 내부·표시 시 반올림)로 KPI=원장=일별누적 끗수 일치.
- 검증: 계약부채 114,778원·달 잔량 3,555 = DB = 이카운트 장부 일치. 커밋 `6c1e789`→`1505959`+금회.
